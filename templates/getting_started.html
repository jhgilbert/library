KEEP_OUT

JANE SAY:

Stay away from KEEP_OUT, hungry bear, and dark forest.

Crog visit KEEP_OUT once. Crog never return.

This one is somewhat self-explanatory, but you know what, I'm not the boss of you. If you want to play it safe, keep out. If you're feeling adventurous, you can install apps into this directory after downloading them from Portable Apps, and then add shortcuts to those apps onto your environment desktop.

The truth is that everything on the disk is in KEEP_OUT, so all of the other shortcuts on your environment desktop will send you into that folder anyway. Just know that altering some of the files in KEEP_OUT will break your environment.

Lucky for you, you'll have lots of instructions on which files to interact with.

Library

JANE SAY:

Library tell you how to catch tiger by tail. Mmm, delicious tiger.

You must be already a bit familiar with what the Library does, if you're reading this page! The Library contains a bunch of training materials for you to read, all accessible from one index.html page that you can drag into your browser..

What you may not have realized is that the Library is magic -- it has the capability to sync with my version, which means you can update it with fresh or corrected material after I add it on my end. We'll get into how to do that a bit later.

This is your code editor -- you'll use it to edit website files, Ruby programs, and anything else that needs editing. It supports multiple languages. Go ahead, double-click on it to pop open a new file.

Your Projects

JANE SAY:

Your Projects like big thick leather pouch made from buffalo scrotum.

Put all code in big leather buffalo scrotum for safest keeping!

You should keep all of your work in Your Projects -- not only that, but you should keep all of your work within one of the directories that's already in Your Projects. This is important for reasons that will become clearer to you when you understand more about version control.

That means that everything you do should go into web_projects, ruby_projects, and so on.

It's okay to make subfolders within those folders -- like putting a my_first_website directory in web_projects. You can, and should, organize your projects into folders when they're made up of more than one file.

Terminal

JANE SAY:

Using Terminal like hunting fish with sharp rock.

Terminal take some time to learn.

Terminal is a topic in and of itself, and you'll spend several days, if not weeks, just learning how to navigate it. In fact, we'll cover the basics in the next section.

For now, here's what you need to know about Terminal:

  • It allows you to quickly navigate the file system of your computer
  • You can use Terminal to create lots of files very rapidly, search files, and so on
  • It allows you to carry out technical tasks that aren't possible otherwise, such as starting a web server
  • It is often a developer's main method of communication with a Linux server (what the big kids use)
  • You will use it to run Ruby programs
  • You will use it to set up and query a database

Terminal has an entire section dedicated to it. When you're ready, come on in -- the water is freezing. But you won't die. (Probably.)
